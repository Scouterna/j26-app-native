# iOS build setup

How to get the J26 iOS app building, running, and sending push notifications —
and eventually published to the App Store. The app is a `WKWebView` wrapper
(generated by PWABuilder) that loads the J26 PWA and adds native push, geolocation,
universal links, and pull-to-refresh.

> **Requires a Mac with Xcode.** The repo edits below are done, but the build
> itself, CocoaPods, signing, and the Apple/Firebase setup can only be done on macOS.

## Current configuration

| Setting | Value | Where |
|---|---|---|
| Bundle ID | `se.j26.app` | `J26.xcodeproj/project.pbxproj` |
| Loads URL | `https://app.dev.j26.se` (**dev** backend) | `J26/Settings.swift` |
| Auth origin | `dev.id.scouterna.se` | `J26/Settings.swift`, `J26/Info.plist` (`WKAppBoundDomains`) |
| Associated domains | `app.dev.j26.se` | `J26/Entitlements/Entitlements.plist` |
| `aps-environment` | `development` | `J26/Entitlements/Entitlements.plist` |
| Push | Firebase Cloud Messaging (`Firebase/Messaging` 12.10.0) | `Podfile` |
| Deployment target | iOS 15.0 | `Podfile`, `project.pbxproj` |
| Dev team / provisioning | **not set** | `project.pbxproj` |

## Already done (in this branch)

- Bundle ID changed `se.j26.dev.app` → `se.j26.app` (Debug + Release) to match the
  registered Apple app.
- `FirebaseApp.configure()` enabled in `J26/AppDelegate.swift`.
- `aps-environment` set to `development` for on-device push testing.

The push plumbing is already wired: Firebase method swizzling is enabled (default),
so the APNs → FCM token hand-off is automatic. The web bridge
(`handleFCMToken`, `sendPushToWebView`, topic subscribe/unsubscribe) lives in
`J26/PushNotifications.swift`. `registerForRemoteNotifications()` is called from the
in-app permission flow.

## Steps remaining (on the Mac)

### 1. Apple Developer Program (developer.apple.com)
- [ ] Confirm App ID **`se.j26.app`** exists.
- [ ] Enable the **Push Notifications** capability on the App ID.
- [ ] Enable the **Associated Domains** capability on the App ID.
- [ ] Create an **APNs Authentication Key** (`.p8`). Note the **Key ID** and **Team ID**
      (one key covers both sandbox and production).

### 2. Firebase (console.firebase.google.com)
- [ ] Add an iOS app to the Firebase project with bundle ID **`se.j26.app`**.
- [ ] Cloud Messaging → upload the APNs `.p8` key (with Key ID + Team ID).
- [ ] Download **`GoogleService-Info.plist`** and place it at
      **`ios/J26/GoogleService-Info.plist`**.
      - The build will fail without it now that `FirebaseApp.configure()` is active.
      - It is **gitignored** on purpose (it's per-environment config / a secret) —
        it stays out of git and must be added manually on each machine.

### 3. Build (Xcode)
- [ ] `cd ios && pod install` (there is no `Pods/` dir yet; `Podfile.lock` pins versions).
- [ ] Open **`J26.xcworkspace`** — *not* `J26.xcodeproj`.
- [ ] Signing & Capabilities → select your **Development Team** (currently empty),
      keep **Automatic** signing.
- [ ] Verify these capabilities are present: **Push Notifications**,
      **Associated Domains**, **Background Modes → Remote notifications**.
- [ ] Build & run on a **real device** (push does not work on the Simulator).

### 4. Verify notifications
- [ ] Trigger the in-app notification permission prompt.
- [ ] Confirm the FCM token reaches the web view (the `push-token` event).
- [ ] Send a test message from Firebase Console → Cloud Messaging to that token.
- [ ] Test all three paths: foreground, background, and tap-to-open.

### 5. Web / backend coordination (not in this repo)
- [ ] The **apple-app-site-association** file served at `app.dev.j26.se` must list the
      App ID `<TeamID>.se.j26.app` for universal links / web credentials to work.

## Before App Store submission

- [ ] Set `aps-environment` back to **`production`** in `J26/Entitlements/Entitlements.plist`
      (sandbox tokens won't receive production pushes, and vice versa).
- [ ] Point the app at the **production** PWA — update `J26/Settings.swift`,
      `J26/Info.plist` (`WKAppBoundDomains`), and `J26/Entitlements/Entitlements.plist`
      (associated domains) to the prod hosts (e.g. `app.j26.se` / `id.scouterna.se`).
      Cleanest approach: a separate **Dev** / **Prod** build configuration + scheme,
      each with its own URLs and `GoogleService-Info.plist`.
- [ ] Set real `MARKETING_VERSION` and `CURRENT_PROJECT_VERSION` (both are `1` now).
- [ ] Decide on **Mac Catalyst**: the project currently has `SUPPORTS_MACCATALYST = YES`
      and device family `1,2,6`, and the entitlements carry macOS App Sandbox keys.
      Drop Catalyst support if you only want an iOS app (simplifies signing).
- [ ] Create the App Store Connect record, archive, upload, test via TestFlight, submit.
